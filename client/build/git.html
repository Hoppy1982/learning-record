Git Commands

This is a record of git commands with examples of how Iâ€™ve used them. Placeholders have been used where further information will be inputted at a later date. The main drive for this page came from a desire to better understand the broad topic that version control with git is.

status

Shows current branch, untracked files in red, tracked files in green. Also shows whether the local and remote branches are in sync.

$ git status

checkout

Switches between branches / commits.

$ git checkout branchName

To create then checkou a branch.

$ git checkout -b newBranchName

diff

To see the differences between two branches / commits.

$ git diff branchOne branchTwo

Just see which files are different.

$ git diff --name-only branchOne branchTwo

View difference between a local branch and remote

$ git diff branchOne origin/branchTwo

log

To view commit history. 'q' to exit from the log back into bash.

Use this to get commit names if you need to checkout or revert to them.

$ git log

Short pretty format

$ git log --pretty=short

remote

To see the remote url

$ git remote -v

add

To add all modified files to staging area

$ git add .

To add an individual file to staging area

$ git add /examplePath/exampleFile.js

commit

Creates a commit on current branch.

If the -m flag is not used default text editor will open.

$ git commit -m "this is a commit message"

Edit to message of the most recent commit.

$ git commit --ammend -m "edited commit message"

branch

To view all branches and current branch.

$ git branch

To create a new branch from the current branch.

$ git branch newBranchName

set-upstream

To set the remote for current branch.

$ git branch --set-upstream-to=origin/branchName

push

To push current branch up to the set remote.

$ git push

To set the remote and push to it.

$ git push -u origin branchName

$ git push origin +branchName
